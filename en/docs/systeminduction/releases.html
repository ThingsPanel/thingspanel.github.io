ThingsPanel-Ver0.3.0​

What's new​

  • Added Redis as a caching module
  • Added RBAC permission management module, using the Casbin permission framework, and the permission granularity is specific to buttons or interfaces
  • Added a role management page
  • Added permission management page
  • The backend added support for Docker environment variables
  • Simplified deployment and increased adaptation to HUAWEI CLOUD/Alibaba Cloud
  • Added access rule engine and data forwarding function
  • The service-device grouping-device module has been rewritten to support unlimited device grouping
  • Added log of operation device, support query manual operation and automatic operation records

Optimization and fixing​

  • Optimized first login loading speed
  • Optimized the overall UI
  • Optimized indexing and sorting of some tables to improve the fluency of operations
  • The standard of stress testing has been raised and various aspects of performance and structure have been optimized
  • Optimized automated alerting and control strategies
  • User added features and pages rewritten
  • Optimized the alarm information page

ThingsPanel-Ver0.2.0​

This time new​

  • Added the main menu of system settings, support for replacing all logos and system names on the system
  • Added the device properties feature to populate the device location to locate the device in the business visualization
  • Optimized the TCP protocol and added a device reset API interface
  • Added an external extension interface：query device history data based on device id
  • Added external extension interface：provides an interface for input device data and pictures
  • Added a picture preview function to data management and fixed the time not showing defect
  • Added the external extension interface：query the current value of all devices and devices under the service according to the service id
  • Added external extension interface：query the current value of the device based on the device id
  • Added the ability to export data to excel
  • A log module has been added in the background, which can configure the number of split lines, 8 log levels, and log saving time in the app.conf configuration file; sql logging is at the info level
  • Alarm information supports service ID, asset ID, and device ID retrieval
  • Added dynamic loading logo and changed the loading style

Defects in this optimization and fix​

  • Fixed an issue where automated alerts or associated events caused service downtime
  • Optimized the non-switching issue of switching assets in the visualization and optimized the switching loading speed
  • Optimized the issue of long loading time of login pages
  • Optimized the issue where the line breaks of chart fields in application management can make the box taller
  • Fixed a bug where the temperature and humidity visualization plug-in graph could not be loaded
  • The title and layout of all plugins have been optimized
  • Optimized the configuration of the front-end packaging deployment part, which is coordinated with docker one-click deployment
  • Optimized the docker version, supporting the update of the front-end code, front-end code and nginx configuration on the docker version
  • Optimized the retrieval function of data management, fixed some bugs, and enhanced the user experience;
  • Optimized the editing business in the business and reduced the display delay; Partial names have been modified
