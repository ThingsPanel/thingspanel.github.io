数据转发

设备使用device/attributes或gateway/attributes主题将属性上报至平台，在规则引擎中可以通过订阅attributes/relaying/{AccessToken}可获取设备上报的属性（需要使用创建设备时候生成的认证信息登录mqtt客户端-与设备登录的认证一致），然后将数据处理后以各种方式转发出去。

1. Description​

Based on a visual stream programming tool, data forwarding to devices can be implemented very easily.

2. Procedure​

1). Create a forwarding rule​

Rule Engine -> Data Forwarding -> Create Forwarding Rule
Click theCreate Forwarding Rulebutton and fill in the custom rule name.

2). Edit​

Rule Engine -> Data Forwarding -> Edit
Once the rule is created, you can click theEditbutton to enter the visual flow programming tool and edit the node.
Details of how the streaming programming tool works can be found in： rules engine like graph：

Once you're done editing, don't forget to clickSave and run the, otherwise the rule won't take effect.

3). Start/pause​

Rule Engine -> Data Forwarding -> Launch/Pause
When we edit a rule in the stream programming tool and save it, we can do it directly in ThingsPanelstart the/Pause, you no longer need to open the streaming programming tool.
